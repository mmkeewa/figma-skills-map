Связки и подводные камни

Токены → компонент

Порядок обязателен: cc-figma-tokens → cc-figma-component. Пока в файле нет коллекций Primitives и Semantic, компонент не соберётся.

Аудит → починка

audit-design-system → fix-design-system-finding. Первый находит дрейф и приоритизирует, второй чинит конкретную находку.

Три пути «код → Figma»

generate-design — по дизайн-системе; edit-figma-design — из текста; prototype-to-figma — из прототипа; generate-library — компоненты.

/multi-agent зовётся иначе

На карточке Figma это /multi-agent, но внутреннее имя — figma-augment-parallel. Вызывать нужно по нему.

create-voice — часть uSpec

Умеет рендерить a11y-спеку, но на вход хочет component-.md из системы uSpec. Полный набор — npx uspec-skills init.

Две синхронизации токенов

sync-figma-token — двусторонняя код↔Figma с дрейфом; cc-figma-tokens — генерация из контракта в одну сторону.
